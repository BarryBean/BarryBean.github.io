白嫖 github 速度慢，还是自己的服务器用起来好呀。

白嫖 github 虽然美滋滋，但一是速度有点慢，二是每次更新都占提交记录，所以还是得部署在自己的云服务器上才香。

注：腾讯云、阿里云、华为云都有学生优惠。

前期准备

  1. 一台腾讯云服务器
  2. 本地的 hexo 博客

云服务器系统为 CentOS 7.4 64位，登录并进入服务器的命令行界面。

配置git

  1. 安装依赖库和编译工具

    • 安装依赖库：
    yum install curl-devel expat-devel gettext-devel openssl-devel zlib-devel
    • 安装编译工具：
    yum install gcc perl-ExtUtils-MakeMaker package

  2. 下载 git

    • 选择目录存放下载的 git 安装包。以/usr/local/src 为例。
    cd /usr/local/src
    • 去 git官网 找一个稳定的源码包，下载到 /usr/local/src 里
    wget https://www.kernel.org/pub/software/scm/git/git-2.16.2.tar.gz

  3. 解压编译

    • 在当前目录下解压 git-2.16.2.tar.gz 
    tar -zvxf git-2.16.2.tar.gz
    • 进入 git-2.16.2.tar.gz 目录下
    cd git-2.16.2
    • 执行编译
    make all prefix=/usr/local/git
    • 安装 git 到 /usr/local/git 目录下
    make install prefix=/usr/local/git

  4. 配置环境变量

    • 将 git 加入 PATH 目录中
    echo 'export PATH=$PATH:/usr/local/git/bin' >> /etc/bashrc
    • 使 git 环境变量生效
    source /etc/bashrc

  5. 查看 git 版本

    git --version

    如果能查到 git 的版本号，说明安装成功。

  6. 创建 git 仓库，存放网站资源。

    • 在 home/git 的目录下，创建一个名为hexoBlog的仓库，并修改所有权和用户权限。
    mkdir /home/git/
chown -R $USER:$USER /home/git/
chmod -R 755 /home/git/
    • 然后在此目录下，创建一个 git 仓库。
    cd /home/git/
git init --bare hexoBlog.git

  7. 创建一个新的 git 钩子，用于自动部署。

    • 在 /home/git/hexoBlog.git 下，有一个自动生成的 hooks 文件夹。我们需要在里边新建一个新的钩子文件 post-receive。
    vim /home/git/hexoBlog.git/hooks/post-receive
    • 按 i /a键进入编辑模式，在该文件中添加如下代码，指定 Git 的工作树（源代码）和 Git 目录（配置文件等）。
    #!/bin/bash
git --work-tree=/home/hexoBlog --git-dir=/home/git/hexoBlog.git checkout -f

    • 按 Esc 退出编辑模式，输入:wq 保存退出。

    • 添加文件执行权限。

    chmod +x /home/git/hexoBlog.git/hooks/post-receive

至此，git 仓库搭建完成。

配置Nginx

  1. 安装
yum install -y nginx
  1. 启动
service nginx start
  1. 测试 Nginx
wget http://127.0.0.1

出现以下页面说明安装成功。

Connecting to 127.0.0.1:80... connected.
HTTP request sent, awaiting response... 200 OK
Length: 19706 (19K) [text/html]
Saving to: ‘index.html’

100%[==================================>] 19,706      --.-K/s   in 0s      

2020-05-10 19:29:17 (311 MB/s) - ‘index.html’ saved [19706/19706]

  1. 配置 Nginx 托管文件目录

    • 创建 /home/hexoBlog目录，并配置权限。
    mkdir /home/hexoBlog/
chown -R $USER:$USER /home/hexoBlog/
chmod -R 755 /home/hexoBlog/
    • 查看 Nginx 默认配置的安装位置
    nginx -t
    • 修改默认配置，配置就是上一步查到的安装位置
    vim /etc/nginx/nginx.conf
    • 按方向键，找到如下位置
    server {
    listen 80 default_server;
    listen [::]:80 default_server;
    root /home/hexoBlog;    #需要修改

    server_name barrybean.com; #博主的域名，需要修改成对应的域名

    # Load configuration files for the default server block.
    include /etc/nginx/default.d/*.conf;
    location / {
    }
    error_page 404 /404.html;
        location = /40x.html {
    }
    • 按i键进入插入模式，将其中的 root 值改为 /home/hexoBlog （刚才创建的托管仓库目录）。
      将 server_name 值改成自己相应的域名地址。

  2. 重启 Nginx 服务

    service nginx restart

至此，服务器端配置结束。

hexo本地配置

  1. 设置 hexo 配置文件
    • 打开 hexo 的站点配置文件，修改如下。
deploy:
  type: git
  repo: root@xx.xx.xx.xx:/home/git/hexoBlog  //xx.xx.xx.xx为服务器地址
  branch: master
  1. 重新生成博客，并部署。
hexo clean
hexo generate
hexo deploy

此时就能通过公网 ip，访问 hexo 博客。想自定义域名就得注册域名 + 备案啦。
